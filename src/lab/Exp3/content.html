Theory

Theory

Arithmetic operators:

The + and - can be used as either unary (-a) or binary (a-b) operators. The modulus operator is not allowed for real data type variables. Other arithmetic operators can be used with all data types. For the modulus operator, the result takes the sign of the first operand. Following table gives the list of Arithmetic operators.

   Description   Operator
    Addition      a + b
    Subtraction       a - b
    Multiplication       a * b
     Division       a / b
     Modulus       a % b


Syntax

module a(clk,a,b,c,d,e);

input clk;

input [7:0] a,b;

output reg[15:0] c,d,e,f,g;

always @(posedge clk)

begin

c=a+b;

d=a-b;

e=a*b;

end

endmodule


For more detail see the sample program number 1.

Comparison/Relational operators:

The relational operators are used to compare expressions. The value returned by the relational operators is 0 if the expression evaluates to false and 1 if expression evaluates to true. Following table gives the list of relational operators.

          Description
Operator
  Equal to(Logical equality)   a==b
  Not equal to(Logical inequality)    a!=b
  Equal to(Case equality)   a===b
  Not equal to(Case inequality)   a!==b
  Greater than   a>b
  Less than   a<b
  Greater than or equal to   a>=b
  Less than or equal to   a<=b


Syntax

module rela_op(clk,a,b);  

input clk;  

output reg a,b;  

always @(posedge clk)  

begin  

a= 4'b1110;    //14 

b= 4'b0110;    //5  

$display( "%b", a < b );//false - 0  

$display( "%b", a > b );//true - 1  

$display( "%b", a <= b );//false - 0  

$display( "%b", a >= 10 );//true - 1  

$display( "%b", a == 4'b1100 );//true - 1 

$display( "%b", a != 4b1100 );//false-0  

end

endmodule


For more detail see the sample program number 2 and 3.

Logical operators:

The logical operators are used to connect expressions.The result for these operators is 0 (when false), and 1 (when true). The negation operator (!) turns a nonzero or true value of the operand into 0, zero or false value into 1. Following table gives the list of Logical operators.

Description Operator
  Logical negation   !a
  Logical AND   a&&b
  Logical OR   a||b


Syntax 
module logical_op(clk,a,b);
input clk;
output reg a, b;
always @(posedge clk)
begin
a = 4'b1100;
b = 4'b0000;
$display( "%b", !a );// false - 0
$display( "%b", a && b );// false - 0
$display( "%b", a || b );// true - 1
end
endmodule


For more detail see the sample program number 3.

Bitwise operators:

Bitwise operators operate on the bits of the operand or operands. Following table gives the list of Bitwise operators.

   Description   Operator
 Bitwise NOT  ~a
 Bitwise AND  a & b
 Bitwise OR  a | b
 Bitwise XOR

 a^b
 Bitwise left shift  a << b
 Bitwise right shift  a >> b


Syntax 
module bitwise_op(clk,a,b,c,d);
input clk;
output reg a,b,c,d;
always @(posedge clk)
begin
a = 4'b1100;
b = 4'b0000;
c = 8'b1010xzxz;
d = 8'b10101011;
$display( "%b", c & d );// =8'b100000xx
$display( "%b", c | d );// =8'b1011xx11
$display( "%b", c ^ d );// =8'b0011xxxx
$display( "%b", c ~^ d );// =8'b1100xxxx
$display( "%b", c ~ d );// =8'b0101xxxx
$display( "%b", a << 3 );// =4'b1000
$display( "%b", a >> 3 );// =4'b0001
end
endmodule

Procedure and Demo

 
Steps to perform experiment:

1. Start simulator by clicking on the Simulation tab.
2. Click on the link to enter the simulator.
3. Login to your virtual Lab account.
4. First of all select a device from Device tab.(e.g. Device->Spartan 3E)
5. Then select an Experiment to perform from list.
6. After filling the necessary fields click on Submit button.
7. Here you will get answer to verify answer from simulator.
8. Now click on Generate Verilog Code button to generate the code.
9. Load the generated program by clicking on Load Program tab.
10. Enter the file name without extension.
11. Click on Save On Server button to save program on your workspace at server.
12. Now click on Compile tab to compile your code on server.
13. After successful compilation, click on ok to generate the test bench.
14. Click on checkbox to select as output registers and select its type as output from list.
15. Enter data width (MSB=7 and LSB=0 for 8-bit output register).
16. Click on Submit button.
17. Click on Submit Test Bench button to submit your test bench.
18. Click on Close This Window button.
19. Click on Execute tab which is under Compile tab.
20. Click on Timing Analysis tab to view Timing Analysis of input program.
21. For Resource Utilization Report click on Resource Utilization Report tab.
22. Your programs are saved on your workspace at server, which you can find under Your Programs tab.
23. If you want to write your own code Go to File-> New and repeat the procedure from step 10 to 21.
